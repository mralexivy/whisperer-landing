The annoying thing about dictation is that code and conversation need completely different handling. Saying "open paren" should give you `(` in your IDE, but "(open paren)" in your email. Most tools just pick one behavior and stick with it.

Whisperer switches automatically based on which app you're in.

## How It Works

You create a profile and assign it to apps. Whisperer checks which app is in front and switches to the right profile. No manual toggling.

### Defaults

Out of the box, everything uses natural language mode. With Pro Pack, you can set up custom profiles:

| App | Suggested Mode | Behavior |
|-----|---------------|----------|
| VS Code | Code Mode | Symbols by voice, casing commands, no autocorrection |
| Cursor | Code Mode | Same as VS Code |
| Terminal | Code Mode | Handles dashes, dots, paths correctly |
| JetBrains IDEs | Code Mode | Full IDE support |
| Slack | Chat style | Casual, lowercase-friendly, natural flow |
| Gmail | Email style | Proper sentences, natural punctuation |
| Apple Notes | Natural language | Standard dictation |
| Safari/Chrome | Natural language | Web forms, search queries |

## Setting Up Profiles

<StepList>
<Step num="1" title="Open Whisperer from the menu bar">
Click the Whisperer icon in your menu bar
</Step>

<Step num="2" title="Go to Settings > Profiles">
Navigate to the Profiles section in Settings
</Step>

<Step num="3" title="Click Add Profile">
Create a new profile for your app
</Step>

<Step num="4" title="Name your profile">
Give it a descriptive name (e.g., "IDE - Code Mode")
</Step>

<Step num="5" title="Select the mode">
Choose Code Mode or Natural Language
</Step>

<Step num="6" title="Assign apps">
Click **Add App** and select from your Applications
</Step>
</StepList>

## Real Examples

### Morning Standup (Slack)

Open Slack, dictate your update:

<VoiceDemo voice="Good morning. Yesterday I finished the API migration and opened a PR. Today I'm working on the onboarding flow redesign. No blockers." output="Good morning. Yesterday I finished the API migration and opened a PR. Today I'm working on the onboarding flow redesign. No blockers." lang="Slack" />

Clean, casual. Good for Slack.

### Coding (VS Code)

Switch to VS Code, Code Mode kicks in:

<VoiceDemo voice="async function camel case fetch user data open paren user id colon string close paren colon promise" output="async function fetchUserData(userId: string): Promise" lang="TypeScript" />

### Client Email (Gmail)

Open Gmail, get proper sentences:

<VoiceDemo voice="Hi Michael. Thanks for the feedback on the prototype. I've incorporated your suggestions and the updated version is attached. Let me know if you'd like to schedule a walkthrough." output="Hi Michael. Thanks for the feedback on the prototype. I've incorporated your suggestions and the updated version is attached. Let me know if you'd like to schedule a walkthrough." lang="Email" />

### Terminal

Code Mode handles commands too:

<VoiceDemo voice="kubectl get pods dash n production dash o wide" output="kubectl get pods -n production -o wide" lang="Bash" />

None of this requires any manual switching. It just happens.
