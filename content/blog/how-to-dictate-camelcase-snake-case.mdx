Naming conventions are why most dictation tools are useless for code. Say "get user name" and you get "get user name" with spaces. Not `getUserName`. Not `get_user_name`. Just words.

Whisperer's Code Mode fixes this. Simple commands, correct output.

## The Commands

Four casing commands. Say the command, then your words:

### camelCase

<VoiceDemo voice='"camel case get user profile"' output="getUserProfile" lang="JavaScript" />

First word lowercase, rest capitalized, no spaces.

### snake_case

<VoiceDemo voice='"snake case get user profile"' output="get_user_profile" lang="Python" />

All lowercase with underscores.

### PascalCase

<VoiceDemo voice='"pascal case user service"' output="UserService" lang="TypeScript" />

Every word capitalized. Class names, mostly.

### CONSTANT_CASE

<VoiceDemo voice='"constant case max retry count"' output="MAX_RETRY_COUNT" lang="Any" />

All caps, underscores. Constants.

## When to Use What

| Convention | Use | Languages |
|-----------|------------|-----------|
| camelCase | Variables, functions | JavaScript, TypeScript, Java |
| snake_case | Variables, functions | Python, Rust, Ruby |
| PascalCase | Classes, types | Most |
| CONSTANT_CASE | Constants, env vars | All |
